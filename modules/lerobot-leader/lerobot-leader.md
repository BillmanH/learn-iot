# lerobot-leader Module Plan (revised)

## Overview

A **local Python application** running directly on the **Thinkstation (Windows, AKS EE host)**.
It reads joint positions from an SO101 leader arm connected via USB serial (Windows COM port)
and publishes them to the AIO MQTT broker over the network using the existing
`publiclistener` (port 1883, no TLS, no auth).

No container. No Kubernetes pod. Runs with `uv run python app.py` on the Windows machine.

---

## Why Not a Pod

The Thinkstation runs AKS Edge Essentials. AKS EE runs a Linux VM on the Windows host, so
Kubernetes pods have no direct access to Windows USB/COM devices. Passing a COM port into a
Linux pod requires complex USB/IP bridging that isn't worth it for a demo. Running directly
on Windows is simpler and more reliable.

---

## Architecture

```
[SO101 Leader Arm]
     USB Serial (COM port, e.g. COM3)
          |
    [app.py - local Python process on Windows]
     uv run python app.py
          |
     MQTT publish (plain TCP, no auth)
          |
  [publiclistener - BrokerListener LoadBalancer]
     <loadbalancer-ip>:1883
          |
  [AIO Internal Broker]
          |
     (future) dataflow -> lower environment
          |
     (future) lerobot-follower module subscribes
```

The `publiclistener` is already configured in `operations/publiclistener.yaml`:
- Port `1883`, no TLS, no authentication
- `serviceType: LoadBalancer` - AKS EE assigns it a local IP on the Windows host
- Find the IP with: `kubectl get service publiclistener -n azure-iot-operations`

## Key Design Decisions

### 1. publiclistener for External MQTT Access

The `publiclistener` BrokerListener is already in the repo at `operations/publiclistener.yaml`.
It exposes the AIO broker on port 1883 as a LoadBalancer service — no TLS, no auth.

Before running the app, apply the listener if not already active and get its IP:

```powershell
kubectl apply -f operations/publiclistener.yaml
kubectl get service publiclistener -n azure-iot-operations
# Note the EXTERNAL-IP - this goes in local_config.yaml as mqtt_broker
```

On AKS EE, the LoadBalancer IP is typically in the `192.168.0.x` range assigned
to the AKS EE virtual network on the Windows host.

### 2. Windows COM Port (not /dev/ttyACM*)

On Windows the SO101 leader arm shows up as a COM port (e.g., `COM3`).
Find it in Device Manager → Ports (COM & LPT) when the arm is plugged in.

The `feetech-servo-sdk` and `lerobot_edge` use `pyserial` under the hood, which
supports Windows COM ports natively — just pass `"COM3"` instead of `"/dev/ttyACM0"`.

Set the port in `local_config.yaml` before running.

### 3. lerobot-edge as a Vendored Local Package

Same as before — `src/lerobot_edge/` vendored from `BillmanH/lerobot@local_nuc`.
UV installs it from the local path via `pyproject.toml`.

To refresh from your fork:
```powershell
cd modules/lerobot-leader
git clone --no-local --filter=blob:none --sparse -b local_nuc https://github.com/BillmanH/lerobot.git _tmp
cd _tmp ; git sparse-checkout set --skip-checks lerobot-edge/src
Remove-Item -Recurse -Force ..\src
Copy-Item -Recurse -Force lerobot-edge\src ..\src
cd .. ; Remove-Item -Recurse -Force _tmp
```

### 4. Calibration Workflow

Calibration runs directly on the Windows machine before the first run.
The output JSON is saved locally and checked into `calibration/`.

```powershell
# From the modules/lerobot-leader directory
uv run python scripts/03_calibrate_leader.py
# Follow prompts to move arm through range
# Saves: calibration/my_awesome_leader_arm.json
```

The `app.py` sets `HF_LEROBOT_CALIBRATION` to point at the local `calibration/` directory,
so no environment variable setup is needed — it just works from the repo directory.

### 5. Only Leader-Side Code

Same as before — only `SO101Leader` and `SO101LeaderConfig`, no follower.

```python
while True:
    action = leader.get_action()
    publish_to_mqtt(action)
    time.sleep(1 / LOOP_HZ)
```

### 6. MQTT Message Format

Topic: `robot/leader/joint_positions`

```json
{
  "timestamp": "2026-04-29T14:23:01.123456+00:00",
  "leader_id": "my_awesome_leader_arm",
  "sequence": 12345,
  "joints": {
    "shoulder_pan": 1024.0,
    "shoulder_lift": 2048.0,
    "elbow_flex": 1500.0,
    "wrist_flex": 1800.0,
    "wrist_roll": 512.0,
    "gripper": 800.0
  },
  "loop_hz": 20
}
```

### 7. No Docker, No Kubernetes Manifest

This module does **not** have a `Dockerfile` or `deployment.yaml`.
It runs as a plain local process:

```powershell
cd modules/lerobot-leader
uv run python app.py
```

## Configuration

`local_config.yaml` — edit this before running:

| Key                | Default                  | Description                                          |
|--------------------|--------------------------|------------------------------------------------------|
| `leader_port`      | `COM3`                   | Windows COM port for leader arm (check Device Manager) |
| `leader_id`        | `my_awesome_leader_arm`  | Must match calibration JSON filename                 |
| `calibration_dir`  | `./calibration`          | Relative to working directory                        |
| `loop_hz`          | `20`                     | Publish rate — edit and restart to tune              |
| `mqtt_broker`      | `192.168.0.x`            | LoadBalancer IP from `kubectl get svc publiclistener -n azure-iot-operations` |
| `mqtt_port`        | `1883`                   | publiclistener port (plain TCP, no TLS)              |
| `mqtt_topic`       | `robot/leader/joint_positions` |                                               |
| `mqtt_qos`         | `0`                      | 0=fire-and-forget, 1=at-least-once                   |

## Pre-Run Checklist

1. Apply the public listener (if not already):
   ```powershell
   kubectl apply -f operations/publiclistener.yaml
   ```
2. Get the broker IP and update `local_config.yaml`:
   ```powershell
   kubectl get service publiclistener -n azure-iot-operations
   ```
3. Find the COM port (Device Manager → Ports) and update `local_config.yaml`
4. Run calibration if not done yet:
   ```powershell
   uv run python scripts/02_setup_motors_leader.py   # one-time motor config
   uv run python scripts/03_calibrate_leader.py       # generates calibration JSON
   uv run python scripts/04_check_calibration.py      # verify
   ```
5. Run the app:
   ```powershell
   uv run python app.py
   ```
